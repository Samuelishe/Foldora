# Product Vision

Foldora - лёгкая Windows 11 утилита для пользовательских иконок папок и legacy Explorer context menu. Foldora не является тяжёлым shell customizer и не должна превращаться в глобальную замену поведения Проводника.

Главная идея: пользователь сам проектирует своё меню создания и оформления папок.

```text
Пользователь сам добавляет пункт меню.
Пользователь сам выбирает произвольную .ico.
Пользователь сам пишет подпись пункта.
Пользователь сам задаёт имя создаваемой папки.
Foldora импортирует иконку в AppData.
Foldora генерирует legacy Explorer context menu из сохранённых entries.
```

Пример текущей плоской модели:

```text
Создать папку
  Череп
  Красная
  Скелет
  Фото
```

Пример будущей вложенной модели:

```text
Создать папку
  Цветные
    Красная
    Синяя
  Готические
    Череп
    Скелет
  Фото
```

## Core Object

Главный MVP-объект - пользовательский `FolderMenuEntry`, а не icon pack.

Entry хранит:

- видимую подпись `DisplayName`;
- имя создаваемой папки `DefaultFolderName`;
- импортированный путь к `.ico` в `%AppData%\Foldora\icons`;
- технический `Id`, не зависящий от подписи;
- состояние `IsEnabled` и порядок `SortOrder`.

Icon packs - будущая возможность импорта/экспорта набора entries. Packs не должны подменять пользовательскую модель MVP и не должны быть обязательным источником меню.

## Product Boundary

Foldora должна делать несколько вещей хорошо:

- импортировать выбранные пользователем `.ico` в AppData;
- хранить понятные JSON-настройки;
- применять и очищать folder icon через `desktop.ini`;
- генерировать безопасное HKCU legacy context menu из сохранённых entries;
- давать пользователю простой WPF-редактор этих entries.

Foldora не должна в MVP:

- патчить системные DLL;
- требовать admin rights;
- писать в HKLM;
- чистить чужие registry keys;
- запускать фоновый сервис;
- перезапускать Explorer без явного отдельного решения;
- быть универсальным shell customizer;
- зависеть от сторонних shell-кастомайзеров.

## Freedom With Safety

Foldora не запрещает пользователю делать странное меню, но запрещает то, что ломает файловую систему, настройки приложения, реестр или Проводник.

Разрешено:

```text
Череп
Череп
Череп
```

Дубликаты `DisplayName` разрешены. Кириллица, emoji и пробелы в `DisplayName` разрешены. Это выбор пользователя.

Foldora запрещает только технически опасные значения и действия:

- invalid Windows folder names;
- reserved Windows names вроде `CON`, `NUL`, `COM1`, `LPT1`;
- control chars;
- битые, пустые, слишком большие или структурно подозрительные `.ico`;
- registry paths вне Foldora-owned roots;
- запись в HKLM;
- удаление чужих shell entries;
- действия вне `%AppData%\Foldora`, если это не явный пользовательский source-файл для import.

## Default State

Отсутствие Foldora menu в Explorer - нормальное дефолтное состояние. После первого запуска settings могут существовать с пустым `CreateFolderMenu.Entries` и `ExplorerIntegrationEnabled = false`.

Пользователь включает integration явно через CLI или будущий WPF UI. Отключение integration не должно удалять entries.
